Playlist
Endpoint

The playlist endpoint is for any playlist related operations.

CONTENTS

Security Requirements

Type Required Description
API Key Your personal API Key assigned to your project, found under account settings
OAuth Access Token Access Token for user account required for this endpoint, please consult the OAuth section of these docs

Operations

A list of all available operations for this endpoint. 

  • POST /playlist/
    Create a new playlist

    Parameter should be passed in the application/json format as part of the request-body.

    Parameter Type Optional Default Description 
    playlist_name
    		 String 
    undefined
    		 The new playlist's name

    Returns

    Will return a playlist object. 

    
{
    "id": "YAk87OrCDJWOexfI6vM5u0QDnUiy6M1o",
    "name": "Playlist Name",
    "description": null,
    "songs":[],
    "creator": "1234567890123",
    "create_date": 1484180121531,
    "followers":[
        "95164972807487488"
    ],
    "artwork": "",
    "private": false,
    "collaborative": false,
    "creatorName":{
        "username": "username",
        "discriminator": "1234"
    }
}

                
  • DELETE /playlist/{playlist_id}
    Delete a playlist

    Parameter should be present in the URL.

    Parameter Type Optional Default Description 
    playlist_id
    		 String 
    undefined
    		 The ID of the playlist to be deleted 
  • PUT /playlist/{playlist_id}/song
    Add a song from youtube or soundcloud source.

    Important

    This operation currently only supports youtube as a source, soundcloud support is not yet integrated.

    Parameter should be present in the URL. They should also be present in the request body in the application/json format.

    Parameter Type Optional Default Description 
    playlist_id
    		 String 
    undefined
    		 The ID of the playlist to which the new song should be added (should be only present in the URL) 
    source
    		 String 
    undefined
    		 The source of the new song; ytb or scd, (should be only present in the request-body) 
    video_id
    		 String 
    undefined
    		 The video ID of the youtube video being added, (should be only present in the request-body) 
    song_id
    		 String 
    undefined
    		 The song ID of the soundcloud song being added, (should be only present in the request-body)

    Returns

    Will return an object on success. 

    
{
    "added": true
}
